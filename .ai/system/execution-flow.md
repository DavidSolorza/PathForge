# Execution Flow

## Standard Feature Flow
1. User request received
2. Planner analyzes and breaks down
3. Architect designs the solution structure
4. Database updates schemas (if needed)
5. Backend implements APIs (if needed)
6. Frontend implements UI
7. UIUX refines the experience
8. Reviewer validates
9. Testing adds/updates tests
10. DevOps updates deployment (if needed)
11. Documentation updates relevant docs

## Quick Fix Flow
For small changes (typo, styling, single component):
1. Frontend (or relevant agent) implements directly
2. Reviewer validates
3. Done

## Bug Fix Flow
1. Reviewer or Testing identifies the bug
2. Frontend/Backend implements the fix
3. Testing verifies the fix
4. Done
