# Naming Conventions

1. Components: PascalCase (UserCard, ProfilePage)
2. Functions: camelCase (getUserData, handleSubmit)
3. Constants: UPPER_SNAKE_CASE for config, camelCase for local constants
4. Files: PascalCase for components, camelCase for utilities/services
5. Types/Interfaces: PascalCase with I prefix for interfaces (optional, be consistent)
6. CSS classes: Tailwind utility classes only
7. Routes: kebab-case (/learning-path, /ai-assistant)
