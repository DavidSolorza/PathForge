# SOLID Principles

1. Single Responsibility: Each module/component has one reason to change
2. Open/Closed: Extend behavior without modifying existing code
3. Liskov Substitution: Subtypes must be substitutable for their base types
4. Interface Segregation: Keep interfaces focused and small
5. Dependency Inversion: Depend on abstractions (DbAdapter), not concretions (LocalStorageService)
